# CreekTable (高阶表格)

`CreekTable` (在底层导出为 `SearchTable`) 是基于 `@ant-design/pro-components` 中 `ProTable` 的深度封装。

在实际的中后台业务开发中，原生的 `ProTable` 往往会遇到诸如：**高度无法自适应导致双滚动条**、**操作列文字被挤压换行**、**长文本溢出撑破布局**、**列宽无法拖拽调整** 等一系列痛点。

`CreekTable` 的核心目标就是**真正做到开箱即用**，它内置了大量的 Hooks 拦截和优化，彻底解决了上述问题，让开发者只需关注业务数据本身。

## 相比原生 ProTable 做的核心增强

### 1. 自动高度计算防双滚动条 (`useTableScrollHeight`)
- **痛点**：原生表格在数据量大时会撑开整个页面，导致浏览器出现外部滚动条，且分页器被挤出屏幕外。
- **解决**：`CreekTable` 会监听窗口 Resize 和内部元素变化，**自动计算页面剩余可用空间**，并将计算出的高度动态赋值给表格的 `scroll.y`。
- **效果**：无论屏幕多大、上方有多少查询条件，表格内容区域都会自动出现内部滚动条，且**分页器始终固定在屏幕底部可视区域**（可通过 `pageFixedBottom={true}` 开启，默认开启）。

### 2. 智能列宽测量与分配 (`useAutoWidthColumns`)
- **痛点**：对于“操作”列或包含动态标签的列，如果没有写死 `width`，在小屏幕或列很多时会被严重挤压换行，导致 UI 错乱。
- **解决**：该特性会在渲染前隐式测量这些列的实际 DOM 宽度，并自动为其分配最小安全宽度。
- **效果**：操作按钮永远在一行内优雅展示，再也不需要手动计算并硬编码 `width: 150`。

### 3. 开箱即用的列宽拖拽 (`useResizableColumns`)
- **痛点**：原生表格不支持列宽拖拽，对于复杂报表来说用户体验较差。
- **解决**：只要开启 `resizable={true}`（默认开启），所有定义了 `width` 的列表头都会自动附带拖拽手柄，用户可自由拉伸列宽。

### 4. 自动全局序号列 (`useIndexColumn`)
- **痛点**：手写序号列很麻烦，尤其是翻页后，第二页的序号经常又从 1 开始算起。
- **解决**：开启 `showIndex={true}`（默认开启）后，会自动在最左侧插入一列“序号”。该序号内置了 `(current - 1) * pageSize + index + 1` 的逻辑，翻页后序号依然全局连续。

### 5. 默认文本溢出截断 (`useEllipsisColumns`)
- **痛点**：用户输入的长串无空格字符会直接撑破表格单元格。
- **解决**：默认开启了全量列的 `ellipsis: true`，长文本会自动显示为 `...`，悬浮时利用 `Tooltip` 完整展示。

### 6. 默认紧凑布局与对齐优化 (`useTableOptions`)
- **痛点**：原生的工具栏在没有 `headerTitle` 时，右侧的操作按钮会显得非常突兀。
- **解决**：内置了状态管理器，默认使用紧凑型 (`size="small"`)，并在没有 Title 时自动将操作栏左对齐，视觉更统一。
