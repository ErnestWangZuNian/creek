# 全局多语言方案 (@creekjs/i18n & extract)

在企业级出海项目中，多语言（i18n）是一个绕不开的难题。传统的 Umi 多语言方案或者原生的 `react-intl` 往往存在以下痛点：
1. **语言切换刷新页面**：Umi 默认的 `setLocale` 会导致整个应用重新加载（白屏闪烁），体验极差。
2. **代码可读性差**：代码里充斥着 `t('menu.home')` 或者 `t('button.submit')`，开发者在看代码时根本不知道这里本来写的是什么中文，极难维护。
3. **手动提取繁琐**：每次写完中文，都要手动去 `en_US.ts` 和 `zh_CN.ts` 里加 key 和翻译，很容易漏掉。

CreekJS 提供了一套**丝滑、智能、响应式**的全局多语言解决方案，由 `@creekjs/i18n` (运行时库) 和 `@creekjs/i18n-extract` (提取工具) 组成。

## 核心优势

- **⚡️ 零刷新切换**：完全基于 React Context (`react-intl`)，切换语言时只有组件树重新渲染，没有任何页面刷新或白屏。
- **📖 中文作为 Key 的回退**：在代码里直接写中文，提取工具会自动将中文作为 `fallback`。例如：`t('pages.demo.niHao', '你好')`，代码可读性满分！
- **🤖 全自动提取**：命令行一键扫描所有文件，自动把写死的中文替换成 `t()` Hook，并自动生成英文配置项。甚至能**自动提取 Umi 的菜单配置**！

## 运行时配置 (@creekjs/i18n)

在 `apps/pc-demo` 中，我们完全抛弃了 Umi 默认的 `locale: {}` 插件，转而自己接管了 `react-intl`。

### 1. 初始化 i18n 实例

我们在 `src/utils/i18n.ts` 中初始化并导出了全局可用的多语言方法：

```typescript
// src/utils/i18n.ts
import { initI18n } from '@creekjs/i18n/react';
import enUS from '../locales/en_US';
import zhCN from '../locales/zh_CN';

const locales: Record<string, Record<string, string>> = {
  zh_CN: zhCN,
  en_US: enUS,
};

initI18n('zh_CN', locales);

// 导出可以在普通 JS/TS 文件中使用的 t，以及在 React 组件中使用的 useT
export { getIntl, getLocale, globalIntl, setLocale, t, useT } from '@creekjs/i18n/react';
```

### 2. 在组件中使用 (零刷新)

为了保证语言切换时组件能实时响应，在 React 组件中我们使用 `useT` hook：

```tsx | pure
import { useT } from '@/utils/i18n';

const DemoComponent = () => {
  const t = useT(); // 注入响应式上下文
  
  // 第二个参数 '新建宠物' 就是中文的 fallback，保证了极高的代码可读性
  return <Button>{t('pages.Demo.index.xinJianChongWu', '新建宠物')}</Button>
}
```

### 3. Umi 菜单的多语言零刷新

Umi 原生的菜单不支持我们自定义的 React Context。为了让菜单也能零刷新切换，我们在 `app.tsx` 的 `layout` 配置中使用了自定义渲染：

```tsx | pure
// apps/pc-demo/src/app.tsx
export const layout: RunTimeLayoutConfig = () => {
  return {
    // ...
    // 禁用 Umi 自带的菜单国际化
    menu: { locale: false },
    // 使用自定义渲染，包裹一层 React 组件来消费 useT
    menuDataRender: (menuData) => {
      return menuData.map((item) => ({
        ...item,
        name: <MenuName name={item.name} path={item.path} />,
      }));
    },
  };
};

// 抽离的 MenuName 组件
const MenuName = ({ name, path }: { name: string; path: string }) => {
  const t = useT();
  // 自动根据路由 path 匹配 key，如果找不到则回退到原始中文 name
  const key = path === '/' ? 'menu.home' : `menu${path.replace(/\//g, '.')}`;
  return <>{t(key, name)}</>;
};
```

## 自动化提取工具 (@creekjs/i18n-extract)

这是本方案的**杀手锏**！开发时你只需要像往常一样直接在代码里写中文：

```tsx | pure
// 开发时：
export const Demo = () => {
  return <div>你好，世界</div>;
};
```

然后运行提取命令：
```bash
pnpm i18n-extract
```

**工具会自动完成以下操作：**
1. 自动引入 `useT` hook。
2. 自动把中文包裹成 `t('pages.Demo.niHaoShiJie', '你好，世界')`（键名默认是文件路径前缀 + 中文拼音）。
3. 自动更新 `src/locales/zh_CN.ts` 和 `en_US.ts`，保留现有翻译，只追加新增词条。
4. **自动读取 `.umirc.ts`** 中的路由配置，把菜单名称也提取到语言包中！

### 配置提取工具

工具通过根目录的 `i18n.config.ts` 进行配置：

```typescript
import { UserConfig } from '@creekjs/i18n-extract';

const config: UserConfig = {
  localeFileType: 'ts',
  localePath: './src/locales/zh_CN.ts',
  path: 'src/**/*.{ts,tsx,js,jsx,vue}', // 扫描范围
  exclude: ['**/node_modules/**', '**/dist/**', '**/.umi/**', '**/locales/**'],
  
  useT: true, // 开启 Hook 模式
  useTDirs: ['pages', 'components'], // 只在这些目录下自动注入 useT
  useTImportStatement: 'import { useT } from "@/utils/i18n"',
  
  extractMenus: '.umirc.ts', // 指定 Umi 路由文件，自动提取菜单中文！
};

export default config;
```

有了这套工具，你的开发心智负担降到了最低：**只管写中文，剩下的交给 CLI 工具**！翻译人员拿到生成的字典文件后，只需填入英文即可，项目立即获得完美的多语言能力。
